# 編集ガイド — 朝報の品質基準(正本)

## 1. 目的と読者

読者は一人。日本のビジネスパーソンで、**日経新聞の代わり**として朝報を読む。
目的は2つ:

1. **キャッチアップ**: 経済・市場・企業・地政学・政策の「今日知っておくべきこと」を10分で把握する
2. **視野の拡大**: 最先端技術やビジネス構造の変化を、背景から理解する(深掘りコーナー)

新聞の紙面を漫然と再現するのではなく、「限られた朝の時間で、判断に効く情報だけを、
構造がわかる形で」届けることを最優先する。

## 2. 紙面構成と分量

| 要素 | 分量 | 役割 |
|---|---|---|
| market_snapshot | 指標4〜8件 | 30秒で市況を掴む |
| executive_summary | 5〜8項目・各90字以内 | 1分で全体像 |
| top_story | 1本 | 今日最も重要な1テーマ |
| sections(固定5) | 各2〜3本 | 分野別の主要ニュース |
| frontier(深掘り) | 1本・1,500〜2,500字 | 視野を広げる解説 |
| insights | 示唆2〜4件+ウォッチ2〜5件 | 明日以降への接続 |
| glossary | 2〜4語 | 学習の補助線 |

本編合計は**約5,000〜6,000字(読了10分)**。詰め込みすぎは読了率を下げる。
迷ったら「本数を増やす」より「1本の質を上げる」を選ぶ。

## 3. ニュースの選定基準

- **重要度(importance)の判定**:
  - `3` 最重要: 市場全体・複数産業・マクロ政策に影響(利上げ、大規模地政学イベント、業界構造を変えるM&A等)。1号に1〜3本まで
  - `2` 重要: 特定産業・主要企業・特定政策に重要
  - `1` 注目: 知っておくと視野が広がる
- 日本の読者視点だが、対象は世界。「日本にどう波及するか」を必ず一段考える
- 単発の株価変動・ゴシップ・憶測記事は拾わない。**構造や流れが変わるニュース**を優先
- 前号までに報じたテーマの続報は「続報」として**差分**(何が新しくわかったか)を書く。
  同じ内容の繰り返しは禁止

## 4. 執筆ルール

- **文体**: 新聞文体(だ・である調)。敬体は使わない
- **見出し(headline)**: 40字以内。体言止め可。具体的な数字・固有名詞を入れる。
  釣り見出し・曖昧な見出し禁止(悪い例:「市場に激震」/ 良い例:「日銀、政策金利1.0%に引き上げ」)
- **facts(何が起きたか)**: 事実のみ。5W1H と数字。1項目1文。出典で確認できることだけを書く
- **why_it_matters(なぜ重要か)**: そのニュースの**構造的な意味**を1〜3文で。
  「大きなニュースだから」ではなく「何がどう変わるから重要か」を書く
- **implications(示唆)**: 事実からの推論。推論であることが伝わる書き方
  (「〜とみられる」「〜の可能性がある」)で、かつ具体的に書く。importance 3 の記事ではほぼ必須
- **数字の作法**: 必ず単位と比較対象を付ける(「2.79%(前日比+0.02pt、30年ぶり高水準)」)。
  数字が確認できないときは書かない(推測で埋めない)

## 5. 事実と推測の区別(最重要ルール)

- facts に推測を混ぜない。推測・分析は why_it_matters / implications に置き、推量表現を使う
- 未確認情報・観測報道は「〜と報じられている(未確認)」と明示する
- 相反する報道がある場合は両論を書く
- **知らないことを埋めるために書かない**。確認できなければ落とす

## 6. ソース(出典)

- 優先順位: **一次情報**(政府・中銀・企業の公式発表、統計)>
  **大手報道**(日経・Reuters・Bloomberg・FT・WSJ・NHK等)> 業界専門メディア > その他
- 各記事に1〜3件。URL は実在するページを指すこと(検索結果から実際に確認したURLのみ使う)
- ペイウォール記事は要約・リンクは可、**本文の複製は禁止**
- SNS の投稿・個人ブログは原則ソースにしない(一次発表者本人の公式発信は例外)

## 7. 深掘り(frontier)の作法

朝報の differentiator。「今日のニュースの背景にある技術・構造」を、前提知識なしで
理解できるように解説する。

- **テーマ選定**: その日・その週のニュースに関連する技術テーマまたはビジネス構造テーマ。
  例: HBMとは何か / 中央銀行のバランスシート縮小の仕組み / 戦争保険とホルムズ海峡 /
  推論チップとNVIDIA支配の行方
- **過去号と重複させない**: `content/issues/` の過去号の frontier.title を確認してから決める
- **構成**(`##` 見出しで3〜5節): 背景(なぜ今このテーマか)→ 仕組み(どう動いているか)→
  プレーヤーと力学 → 展望・論点 → 読者への示唆
- 専門用語は初出で一行説明。図解が必要なほど複雑にしない
- 分量: 1,500〜2,500字。薄く広くではなく、1つの問いに深く答える

## 8. market_snapshot

- 基本セット: 日経平均・TOPIX・USD/JPY・長期金利(10年JGB)・米10年債・原油(WTIまたはBrent)
- 状況に応じて追加: 金、ビットコイン、S&P500、SOX指数など(合計8件まで)
- `as_of` に「いつ時点の値か」を必ず書く(例: 「7月6日 東京市場終値・NY市場3日終値」)
- 騰落の direction: 上昇=up(赤表示)、下落=down(青表示)。日本市場の慣習
- 値が確認できない指標は省略する

## 9. glossary(今日の用語)

- その号の本文で実際に使った専門用語から2〜4個
- 「読者が調べたくなる語」を選ぶ(初出の技術用語、金融用語、制度名)
- 定義は200字以内。可能なら「なぜ今この語が出てくるか」まで一言添える

## 10. 継続性(watchlist の運用)

- `content/watchlist.md` は号をまたぐ編集メモ。毎朝、号を書き終えたら更新する
- 書くこと: ①継続ウォッチ中のテーマと現状 ②次号で確認すべき予定・指標(日付つき)
  ③直近の frontier テーマ一覧(重複回避用)
- 解決・陳腐化した項目は削除する。リストを無限に育てない(20項目以内)
